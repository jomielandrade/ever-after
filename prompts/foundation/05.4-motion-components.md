# Objective

Implement the reusable motion component library for Project Ever After.

The purpose of this prompt is to establish reusable animation primitives that can be shared across the application.

These components should remain independent of any specific feature.

---

# Context

Before making changes, read:

* AGENTS.md
* docs/02-design-system.md
* docs/03-technical-architecture.md
* prompts/foundation/05-shared-components.md

Follow all applicable Cursor rules.

---

# Scope

Implement only the reusable motion components.

Do not implement feature-specific animations.

---

# Files Allowed to Modify

Create files only inside:

components/shared/motion/

Install additional dependencies only if required.

---

# Components

Create:

* FadeIn
* Reveal
* Stagger

Each component should:

* Be reusable
* Accept children
* Accept className
* Support configurable delay and duration where appropriate
* Respect prefers-reduced-motion
* Follow accessibility best practices

Use Framer Motion only if animation primitives are required.

---

# Animation Philosophy

Animations should feel:

* Calm
* Elegant
* Natural
* Slow enough to appreciate
* Never distracting

Prefer:

* Fade
* Opacity
* Gentle translateY
* Soft scale

Avoid:

* Bounce
* Elastic motion
* Excessive rotation
* Flashing
* Aggressive easing

---

# Constraints

Do NOT:

* Animate the Envelope
* Animate Story sections
* Animate Timeline
* Create page transitions
* Create feature-specific motion

Only create reusable animation wrappers.

---

# Acceptance Criteria

The implementation is complete when:

* Motion wrappers are reusable.
* Components are accessible.
* Reduced-motion is respected.
* Components remain independent of application features.

---

# Deliverables

Provide:

1. Components created
2. Props exposed
3. Accessibility considerations
4. Future use cases

---

# Stop Condition

Stop immediately after completing the motion library.

Wait for approval before continuing.
