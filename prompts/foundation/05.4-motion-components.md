# Objective

Implement the reusable motion foundation for Project Ever After.

The goal is to establish a consistent animation system that is elegant, accessible, reusable, and independent of application features.

This prompt creates the shared motion primitives used throughout the project.

---

Follow all applicable Cursor rules.

# Motion Foundation

## transitions.ts

Centralize reusable transition presets.

Examples:

* gentle
* elegant
* slow

Avoid hardcoding transition values inside components.

---

## variants.ts

Centralize reusable animation variants.

Examples:

* fade
* reveal
* stagger

Avoid duplicating animation definitions.

---

## prefers-motion.ts

Create a reusable helper that respects the user's reduced-motion preference.

This helper should be reusable throughout the application.

---

# Motion Components

Create:

* FadeIn
* Reveal
* Stagger

Each component should:

* Be reusable
* Be fully typed
* Accept `children`
* Accept `className` when appropriate
* Accept animation configuration through props where appropriate
* Export its props type
* Define `displayName`

These components should compose the shared variants and transitions rather than redefining them.

---

# Motion Philosophy

Animations should feel:

* Calm
* Elegant
* Romantic
* Intentional

Prefer:

* Opacity
* Gentle translateY
* Soft scale

Avoid:

* Bounce
* Elastic effects
* Flashing
* Excessive rotation
* Distracting motion

Animation should support the content—not compete with it.

---

# Accessibility

Respect `prefers-reduced-motion`.

When reduced motion is enabled:

* Remove unnecessary transforms.
* Minimize animation duration.
* Preserve usability.

Accessibility is required.

---

# Constraints

* Create page transitions.
* Create feature-specific animations.
* Animate the envelope.
* Animate the story.
* Animate the timeline.
* Modify shared layout components.
* Modify typography components.
* Install additional animation libraries.

Use only the `motion` package.
