# Objective

Implement the shared display components for Project Ever After.

These components establish the reusable visual building blocks used throughout the application while remaining independent of any feature.

---

# Context

Before making any changes, read:

* AGENTS.md
* docs/02-design-system.md
* docs/03-technical-architecture.md
* prompts/foundation/05-shared-components.md

Follow all applicable Cursor rules.

---

# Scope

Implement only the shared display components.

Do not create motion components.

Do not implement pages.

Do not implement feature-specific UI.

---

# Files Allowed to Modify

Create files only inside:

components/shared/display/

Create an `index.ts` barrel export.

Update:

* components/shared/index.ts

to export the new display components.

---

# Components

Create:

* Divider
* DecorativeFlourish
* CardFrame

---

# Component Responsibilities

## Divider

A reusable visual separator.

Requirements:

* Horizontal by default
* Support vertical orientation
* Configurable spacing
* Accessible
* Decorative only

---

## DecorativeFlourish

A lightweight decorative accent.

Requirements:

* Reusable
* Theme-aware
* Subtle
* Optional sizing variants

Do not hardcode wedding-specific imagery.

---

## CardFrame

A reusable content container.

Use for:

* Story cards
* Timeline items
* Venue information
* RSVP summaries
* Gallery items

Responsibilities:

* Consistent padding
* Border
* Background
* Radius
* Shadow

Remain content-agnostic.

---

# Component Standards

Every component must:

* Be fully typed
* Accept className overrides
* Accept children when appropriate
* Define displayName
* Export its props type
* Be responsive
* Follow the Design System

---

# Constraints

Do NOT:

* Create loading screens
* Create motion components
* Create feature-specific components
* Modify globals.css
* Install packages

---

# Acceptance Criteria

The implementation is complete when:

* Display components are reusable.
* Components follow the Design System.
* Components remain feature-independent.
* Shared exports are updated.

---

# Deliverables

Provide:

1. Files created
2. Public exports added
3. Purpose of each component

---

# Stop Condition

Stop immediately after completing the display components.

Wait for approval before continuing.
