# Objective

Create the public export surface for the shared component library.

The goal is to provide a single import path for all shared components while keeping the internal folder structure encapsulated.

---

# Context

Before making changes, read:

* AGENTS.md
* docs/03-technical-architecture.md
* prompts/foundation/05-shared-components.md

Follow all applicable Cursor rules.

---

# Scope

Implement only the shared component barrel exports.

Do not modify existing component implementations.

Do not create new UI components.

---

# Files Allowed to Modify

Create:

* components/shared/index.ts

Update only if necessary:

* components/shared/layout/index.ts
* components/shared/typography/index.ts

---

# Requirements

Create a root barrel export that re-exports all shared component categories.

Initially export:

* layout
* typography

The structure should support future categories without requiring changes to consuming code.

Future categories include:

* display
* motion

The public API should remain stable as the library grows.

---

# Export Strategy

Use barrel exports.

Consumers should be able to write:

```tsx
import {
  Container,
  Section,
  PageTitle,
  Body,
  Caption,
} from "@/components/shared";
```

without importing from nested folders.

---

# Constraints

Do NOT:

* Modify component implementations.
* Rename components.
* Move files.
* Create display components.
* Create motion components.
* Introduce new dependencies.

---

# Acceptance Criteria

The implementation is complete when:

* A single public export file exists.
* Layout components are exported.
* Typography components are exported.
* Existing imports continue to work.
* Future component groups can be added easily.

---

# Deliverables

Provide:

1. Files created
2. Public exports added
3. Example import statement

---

# Stop Condition

Stop immediately after creating the shared barrel export.

Wait for approval before continuing.
