# Objective

Implement the shared typography component library for Project Ever After.

These components establish the application's reusable typography system and must remain independent of any specific feature.

---

# Context

Before making changes, read:

* AGENTS.md
* docs/02-design-system.md
* docs/03-technical-architecture.md
* prompts/foundation/05-shared-components.md

Follow all applicable Cursor rules.

---

# Scope

Implement only the shared typography components.

Do not create pages.

Do not create layout components.

Do not create feature-specific UI.

---

# Files Allowed to Modify

Create files only inside:

components/shared/typography/

---

# Components

Create:

* PageTitle
* SectionTitle
* Eyebrow
* Lead
* Body
* Caption

Create an `index.ts` barrel export.

---

# Component Responsibilities

## PageTitle

Use for the main title of a page.

There should generally be only one per page.

---

## SectionTitle

Use for major section headings.

Should establish the primary content hierarchy.

---

## Eyebrow

Use for small uppercase labels displayed above headings.

Examples:

* Our Story
* Wedding Day
* RSVP

Do not hardcode any content.

---

## Lead

Use for introductory paragraphs beneath titles.

Should emphasize readability.

---

## Body

Use for standard paragraphs throughout the application.

Should inherit the project's body typography.

---

## Caption

Use for:

* Metadata
* Dates
* Locations
* Supporting information
* Small descriptive text

---

# Requirements

Every component must:

* Be fully typed
* Accept children
* Accept `className` overrides
* Be responsive
* Follow the Design System
* Remain content-agnostic
* Export its props type
* Define a `displayName`

Use semantic HTML whenever appropriate.

---

# Accessibility

Use semantic elements where appropriate:

* h1
* h2
* p
* span

Do not reduce accessibility by replacing semantic elements unnecessarily.

---

# Constraints

Do NOT:

* Hardcode wedding content
* Create feature-specific styling
* Modify globals.css
* Modify layout components
* Duplicate typography logic
* Install packages

---

# Acceptance Criteria

The implementation is complete when:

* Typography hierarchy is reusable.
* Components are responsive.
* Components are accessible.
* Components are composable.
* Components remain independent of application features.
